javascript知识点总结

发表于 2018-02-28 | 分类于 js | 阅读次数

javaScript是基于原型，面向对象，所类型的动态脚本语言。

主要分三部分：

  • 核心(ECMAScript)
  • 文档对象模型(DOM)
  • 浏览器对象模型(BOM)

JavaScript 有七种内置类型

  • null
  • undefined
  • boolean
  • number
  • string
  • object
  • symbol (es6)

    什么是异步编程，为什么它在JavaScript中很重要？

    同步

    同步编程意味着，除条件和函数调用之外，代码从上到下按顺序执行，阻塞长时间运行的任务，如网络请求和磁盘I / O。

    异步

    异步编程意味着引擎在事件循环中运行。当需要阻塞操作时，请求开始，并且代码保持运行而不阻塞结果。当响应准备就绪时，将触发中断，这将导致运行事件处理程序，控制流将继续执行。这样，一个单独的程序线程可以处理许多并发操作。
    用户界面本质上是异步的，并且花费大部分时间等待用户输入来中断事件循环并触发事件处理程序。
    节点在默认情况下是异步的，这意味着服务器以相同的方式工作，等待网络请求的循环，并在处理第一个请求时接受更多的传入请求。
    这在JavaScript中很重要，因为它非常适合用户界面代码，并且非常有利于服务器的性能。

    什么是双向数据绑定和单向数据流，它们有何不同？

    双向数据

    绑定意味着UI字段会动态绑定到模型数据，以便在UI字段更改时，模型数据会随之更改，反之亦然。

    单向数据流

    数据流的一种方式意味着该模型是真相的唯一来源。用户界面触发消息中的变化，将用户意图指示给模型（或React中的“存储”）。只有模型有权更改应用程序的状态。其效果是数据总是以单一方向流动，这使得它更易于理解。
    数据流的一种方式是确定性的，而双向绑定会导致难以遵循和理解的副作用。
    • React是单向数据流的新规范示例。
    • Angular是一个使用双向绑定的流行框架。
